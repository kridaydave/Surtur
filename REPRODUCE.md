# Surtur Reproduction Guide

This is the one-command path from a clean clone to verified output.

## Quick reproduce

```bash
git clone https://github.com/kridaydave/Surtur.git
cd Surtur
bash reproduce.sh
```

Expected: `ALL CHECKS PASSED` · `ORCHESTRATOR SMOKE TEST PASSED` · `[Eval Smoke] PASSED`.

---

## What "reproduced" means

| Component | Tolerance |
|-----------|-----------|
| Freeze + guard test | Bit-identical |
| Training engine (CPU) | Bit-identical |
| Orchestrator (CPU) | Bit-identical |
| Training engine (GPU) | ±1% on loss/grad norm (CUDA nondeterminism) |
| Eval scores (opt-125m, 5 examples) | Exact match |

For bit-reproducible GPU runs, import `enable_deterministic` from `src/seed.py` before training. Expect a 10–30% throughput penalty.

## Layers pinned

### 1. Environment — `requirements.lock.txt`

Generated by `pip freeze` from the verified working set (2026-07-09). Install with:
```bash
pip install -r requirements.lock.txt
```

Python: 3.12+ (tested on 3.14 CPU, 3.12 Colab).

For GPU:
```bash
pip install torch --index-url https://download.pytorch.org/whl/cu126
```

### 2. Randomness — `src/seed.py`

```python
from seed import set_seed, enable_deterministic
set_seed(42)          # python, numpy, torch (CPU + CUDA)
enable_deterministic() # bit-reproducible CUDA ops (slower)
```

Every run starts with `set_seed(seed)`. The seed is recorded in the config and passed to TRL's `SFTConfig(seed=...)`.

### 3. Data — `data/MANIFEST.json` (SHA-256)

Every dataset has its SHA-256 pinned. `load_verified(path)` refuses to load if the hash changed.

```python
from data_utils import load_verified
load_verified("data/p0_train.jsonl")  # raises on hash mismatch
```

### 4. The exact run — `orchestrator.py`

```bash
cd src && python orchestrator.py
```

Reads `src/config.yaml`, runs arms × seeds, prints a summary table. One command.

### 5. Smoke test — built into `reproduce.sh`

Five steps: install → freeze test → train smoke → orchestrator smoke → eval smoke.

## Hardware assumptions

- **Dev/CPU:** any x86_64 box, ≥4GB RAM. Model: `facebook/opt-125m`.
- **Production/GPU:** single GPU, ≥16GB VRAM (T4/A10/L4). Model: `Qwen/Qwen2.5-1.5B` (1.5B) or `Qwen/Qwen2.5-7B` (7B).
- unsloth is **required** for real GPU runs. Install: `pip install unsloth`.

---

## What's *not* in this package

- The real P0 training data (`data/p0_train.jsonl` is a placeholder). Hash your real data and add it to `data/MANIFEST.json` before running the full experiment.
- The full 4-arm × 5-seed run. That needs a GPU machine and real data; this package is the foundation that makes it reproducible.
- The dashboard (P3).

---

## If a rerun fails

1. **"data hash mismatch"** → your `data/p0_train.jsonl` differs from the pinned version. Either restore the original or update `MANIFEST.json`.
2. **"frozen layer received gradient"** → the freeze + guard bug-check fired. Run `test_freeze.py` alone to debug.
3. **"no module named trl"** → `pip install -r requirements.lock.txt` didn't complete. Check pip version and retry.
4. **CUDA OOM** → drop to `Qwen2.5-1.5B` or reduce `batch_size` in `config.yaml`.

Every smoke test prints the exact line that broke. Read the line above the traceback — that's the layer that drifted.
